# SPEC

## SUMMARY

- Change `kit handoff`, `kit summarize`, and `kit code-review` to copy generated output to the clipboard by default.
- Reserve raw stdout prompt output for explicit `--output-only` usage while keeping `--copy` as an explicit override for `--output-only`.

## PROBLEM

- These three support commands still print their full prompt or output body by default while the rest of Kit's prompt-producing surfaces are now clipboard-first.
- The inconsistent default makes the CLI harder to predict and forces users to remember different copy/paste flows for adjacent commands.
- Users explicitly requested that the remaining commands follow the same default output contract.

## GOALS

- Make `kit handoff`, `kit summarize`, and `kit code-review` copy their generated output to the clipboard by default.
- Preserve `--output-only` as the way to emit raw stdout output.
- Preserve `--copy` as an explicit override for `--output-only`.
- Reuse the shared clipboard-first helper instead of duplicating output logic in each command.
- Update README and command help text to reflect the final behavior.

## NON-GOALS

- Changing the actual prompt or instruction content generated by `kit summarize` or `kit code-review`.
- Redefining the `kit handoff` prompt contract beyond output transport. That behavior is owned by `0011-handoff-document-sync`.
- Changing command arguments, selectors, or workflow guidance.
- Changing commands that already have their own feature specs for this behavior.

## USERS

- Users switching agents with `kit handoff`.
- Users compressing context with `kit summarize`.
- Users pasting review instructions with `kit code-review`.

## SKILLS

## REQUIREMENTS

- `kit handoff`, `kit summarize`, and `kit code-review` must keep their existing command surfaces.
- `kit summarize` and `kit code-review` must keep their generated content.
- `kit handoff` output transport is owned here, while prompt content may evolve under `0011-handoff-document-sync`.
- Those commands must support `--copy` and `--output-only` with the same clipboard-first semantics as the rest of Kit's prompt-producing commands.
- By default, each command must copy the generated output to the clipboard, print an acknowledgement, and not print the output body to stdout.
- With `--output-only`, each command must print the raw output to stdout and only copy when `--copy` is also set.
- The implementation must reuse the shared clipboard-first helper already used by other commands.
- README must document the expanded clipboard-first command set.

## ACCEPTANCE

- Running `kit handoff`, `kit summarize`, or `kit code-review` with default flags copies the generated output to the clipboard, prints an acknowledgement, and does not print the output body.
- Running any of those commands with `--output-only` prints the raw output to stdout.
- Running any of those commands with `--output-only --copy` both prints and copies.
- Help text and README match the shipped behavior.

## EDGE-CASES

- `--output-only --copy` is passed and must both print and copy.
- `kit handoff` is used in project-wide mode and still follows the new default output contract.
- `kit summarize` is used without a feature argument and still follows the new default output contract.
- `kit code-review` is run with no changed files to review and still follows the new default output contract.
